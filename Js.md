### 1. `package.json` = central config

Basic shape (for any Node app):

```json
{
  "name": "myapp",
  "version": "1.0.0",
  "main": "index.js",
  "scripts": {
    "start": "node index.js",
    "build": "echo 'no build step defined'",

    "nodemon": "^3.0.1"

```

**Key fields:**

* `"name"` â†’ app/package name (like artifactId)
* `"version"` â†’ semantic version like Maven
* `"main"` â†’ entry JS file (for libraries / Node apps)
* `"dependencies"` â†’ required in production
* `"devDependencies"` â†’ only needed in dev/test
* `"scripts"` â†’ most important for CI/CD: defines commands you actually run.

**Script usage:**

```bash
npm start         # runs "start"
npm test          # runs "test"
npm run build     # runs "build"
npm run lint      # if script named "lint"
```

CI pipelines **just call these scripts**.

### 2. `node_modules` and `package-lock.json`

* `node_modules/` â†’ downloaded dependencies (like a local Maven repo for that project)
* `package-lock.json` â†’ locks exact versions of dependencies and sub-dependencies.

You must **commit `package-lock.json`** for consistent CI builds.

Install:

```bash

```

### 3. React Environment Variables

React uses **`.env` files** and variables prefixed with `REACT_APP_` (for CRA) or just `VITE_` (for Vite).

Example `.env`:

```env
REACT_APP_API_BASE_URL=https://api.mycompany.com
REACT_APP_FEATURE_X_ENABLED=true
```

Usage in code:

```js

```

In CI/CD you can:

* Inject `.env` file
* Or set env vars before `npm run build`

### 4. React in Jenkins (CI/CD Example)

```groovy

  agent any

  stages {
    stage('Checkout') {
      steps { git 'https://github.com/org/my-react-app.git' }
    }
    stage('Install') {
      steps { sh 'npm install' }
    }
    stage('Test') {
      steps { sh 'npm test -- --watch=false' }

    }
  }
}
```

Then deploy the `build/` folder.
